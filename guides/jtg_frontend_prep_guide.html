JTG Frontend Internship + PPO — The process has 7–9 elimination rounds. Every round counts. CSS gets people eliminated most. JavaScript depth wins offers. DSA is required even for frontend.

Real talk: Multiple candidates were rejected specifically due to weak CSS skills in technical interviews. One reported being eliminated in Round 7 after failing CSS questions. CSS is their filter.

Round 1 — MCQ Assessment
70 questions · 1 hour · Negative marking
Aptitude + Frontend ▼
  • 50 questions: pseudo-code, input/output prediction, language syntax (JS heavy)
  • Remaining: aptitude, OS basics, DBMS, DSA easy-level MCQs
  • Questions like: "What does typeof null return?", "Output of this JS snippet?"
  • HTML5 semantic tags, CSS box model, specificity rules
  • Negative marking — don't guess randomly
Round 2 — Coding Test (DSA)
2 questions · 1 hour · Both must be solved
DSA Required ▼
  • Arrays: third largest element, maximum subarray, missing positive number
  • Linked Lists: swap nodes, reverse in K groups
  • Stack/Queue based problems
  • DP (Dynamic Programming) — easy to medium level
  • Both questions must be solved to advance — partial credit not enough
  • Write comments — shows approach even if code is incomplete
Round 3 — Advance Subjective
5 questions · 70 minutes · Mixed frontend
HTML + CSS + JS ▼
  • Given a webpage screenshot — write HTML & CSS to match it exactly
  • JavaScript form validation with specific functionality
  • Read JS snippet → write correct output (closures, hoisting, async)
  • Balanced parentheses / DSA in JS
  • Platform: Calyxpod
Round 4 — Take-Home Assignment
1 week · Static page only · No frameworks
Critical Round ▼
  • Build a portfolio website from a Figma design provided by JTG
  • Only HTML, CSS, JavaScript — no React, no Bootstrap, no jQuery
  • Must be pixel-perfect to the design
  • Code of conduct enforced — clean, readable, commented code
  • Responsive design expected
  • This is where most candidates get eliminated
Round 5 — MCQ (DSA + Frontend)
Medium difficulty · 1 hour
Mixed ▼
  • DSA + Frontend technology MCQs combined
  • Deeper JS questions — closures, promises, event loop
  • CSS layout specifics — Flexbox, Grid, positioning
  • Only shortlisted students from Round 4 reach here
Round 6 — Live Coding (Screen Share)
2–3 hours · Updates every 20 mins · JS focused
Hardest Round ▼
  • Build a complete website with JavaScript functionality — live, screen shared
  • Interviewer checks progress every 20 minutes
  • Google is allowed for syntax — logic must be your own
  • React component structuring and JS event handling tested here
  • Speed + correctness + code quality all evaluated
Rounds 7–8 — Technical Interviews
In-depth · Face to face · Project based
CSS Killer Round ▼
  • Deep questions on HTML, CSS, JavaScript, Computer Networks
  • CSS is the focus — interviewers specifically judge CSS skill
  • Questions from your assignment: "Why did you write this CSS this way?"
  • JS concepts: closures, this keyword, promises, event delegation
  • Project discussion + live debugging of your own code
  • One experienced candidate was eliminated here due to CSS weakness
Round 9 — HR Interview
45 mins · Personality + Communication
Non-Technical ▼
  • Personality traits, adaptability, communication
  • Why frontend? Why JTG? Future goals?
  • Work environment expectations
  • Straightforward — most candidates pass if they reach here
  • PPO discussion happens here if selected

Strategy: JTG goes depth over breadth. Master each topic completely — especially CSS — before moving on. Do not skip the output-prediction practice. Build at least 3 projects in pure HTML/CSS/JS.

Week 1–2
HTML — Full Depth
Goal: Know every semantic tag, form attribute, and HTML5 feature. Be able to convert any design to clean HTML.
  • All semantic tags: header, nav, main, section, article, aside, footer
  • All form elements + input types + validation attributes
  • Accessibility: alt, aria-label, tabindex, role
  • data-* attributes, meta tags, viewport, charset
  • script defer vs async — memorize the difference
  • Practice: clone 5 different webpage layouts in pure HTML
MDN HTML Guide web.dev/learn/html W3Schools HTML
Week 3–5
CSS — Their Main Filter (Most Time Here)
Goal: Master Flexbox and Grid completely. Understand every positioning rule. Build pixel-perfect layouts from any design.
  • Box model: content, padding, border, margin — draw it from memory
  • Flexbox: every single property (justify-content, align-items, flex-wrap, flex-grow)
  • CSS Grid: grid-template, areas, auto-fit, minmax
  • Positioning: relative, absolute, fixed, sticky — know when to use each
  • Specificity: inline > ID > class > element — calculate any selector
  • Responsive: mobile-first, media queries, viewport units
  • Pseudo-classes: :hover, :nth-child, :not — and pseudo-elements: ::before, ::after
  • Transitions & @keyframes animations
  • Play CSS Diner (selectors) and Grid Garden + Flexbox Froggy daily
  • Practice: rebuild JTG-style portfolio pages from Figma mockups
Kevin Powell YouTube web.dev/learn/css Grid Garden Flexbox Froggy
Week 6–9
JavaScript — Go Very Deep
Goal: Predict output of any JS snippet. Write DOM manipulation, form validation, and async code from scratch under time pressure.
  • Closures — understand and write examples daily
  • Hoisting: var, let, const, function declarations vs expressions
  • this keyword: global, object method, arrow function, bind/call/apply
  • Event Loop: call stack, web APIs, callback queue, microtask queue
  • Promises + async/await: chain them, handle errors, know the order
  • DOM: querySelector, addEventListener, createElement, event delegation
  • ES6+: destructuring, spread, rest, template literals, optional chaining
  • Array methods: map, filter, reduce, find, some, every
  • AJAX / fetch API — write a real API call from scratch
  • localStorage, sessionStorage, cookies — differences
  • Form validation in pure JS — JTG assignment staple
  • Daily: read 3 JS snippets on javascript.info and predict output before running
javascript.info You Don't Know JS freeCodeCamp
Week 10–11
DSA — Just Enough for JTG
Goal: Solve easy-medium array, linked list, and stack problems. Write in JavaScript. Solve at least 30 LeetCode problems.
  • Arrays: two-pointer, sliding window, prefix sum, sorting
  • Linked Lists: reversal, merge, detect cycle, swap nodes
  • Stacks: balanced parentheses, next greater element
  • Dynamic Programming: climbing stairs, coin change (easy DP only)
  • Binary Search: basic + "binary search on answer" pattern
  • Solve problems in JavaScript — JTG expects JS logic even in DSA
  • Target: 30–40 LeetCode Easy/Medium problems, then revise

Week 12–13
Projects + Mock Assignments
Goal: Build 3 complete projects in pure HTML/CSS/JS. Simulate the JTG take-home assignment under real conditions.
  • Build a portfolio site (JTG assignment is exactly this — from Figma)
  • Build a form with full JavaScript validation (name, email, password, etc.)
  • Build a to-do app or weather app using fetch API
  • Practice: take a screenshot of any website → recreate it in 2 hours
  • No frameworks — vanilla HTML, CSS, JS only
  • Code must be clean, commented, and follow BEM or consistent naming
  • Push everything to GitHub — they check your code style
Week 14
CS Fundamentals + Mock Interviews
Goal: Be ready for technical interview questions on networks, browsers, and past project explanations.
  • HTTP vs HTTPS, status codes (200, 404, 500, 301)
  • How browsers render a page (DOM → CSSOM → Render tree)
  • DNS, how the internet works (basic)
  • REST API basics: GET, POST, PUT, DELETE
  • Review all your projects — explain every line of CSS you wrote
  • Do 2–3 mock interviews with a friend asking deep CSS + JS questions
  • Watch JTG interview experience videos on YouTube
