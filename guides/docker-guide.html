Docker Essentials

A quick-reference tutorial for the most commonly used Docker commands, core concepts, and deployment configurations.

Docker DevOps
01 — Images

Working with Images

Images are the blueprints of your application. They are read-only templates that contain everything needed to run your code.

💡
Pro Tip
On Linux, you may need to prefix commands with sudo unless you add your user to the docker group.
CommandDescription
docker pull <image>Download an image from Docker Hub
docker imagesList all downloaded images locally
docker rmi <image_id>Remove a specific image
docker build -t <name> .Build an image from a Dockerfile in the current directory
$docker pull ubuntu:latest
$docker build -t my-app:v1 .
02 — Containers

Running Containers

Containers are the live, running instances of your images.

CommandDescription
docker run <image>Create and start a new container
docker psList all currently running containers
docker ps -aList all containers (running and stopped)
docker stop <container_id>Gracefully stop a running container
docker rm <container_id>Remove a stopped container
docker exec -it <id> shOpen an interactive shell inside a running container
⚠️
Common Flags for Run
-d (Detached mode: run in background)
-p 8080:80 (Port mapping: host_port:container_port)
--name my-nginx (Assign a custom name to the container) 
$docker run -d -p 8080:80 --name web-server nginx
03 — Data Persistence

Volumes

By default, data inside a container is lost when the container is removed. Volumes allow you to persist data permanently on the host machine.

CommandDescription
docker volume create <name>Create a new volume
docker volume lsList all volumes
docker volume rm <name>Delete a volume

To attach a volume when running a container, use the -v flag.

$docker run -d -v my-data:/var/lib/mysql mysql
ℹ️
Bind Mounts
You can also map a specific folder on your computer directly to the container: -v /home/user/app:/app. This is great for local development!
04 — Building

Dockerfile Basics

A Dockerfile is a text document containing all the commands a user could call to assemble an image.

# 1. Use an official runtime as a parent image
FROM node:18-alpine

# 2. Set the working directory
WORKDIR /usr/src/app

# 3. Copy package.json and install dependencies
COPY package*.json ./
RUN npm install

# 4. Copy the rest of your app's source code
COPY . .

# 5. Expose the port the app runs on
EXPOSE 3000

# 6. Define the command to run your app
CMD ["npm", "start"]
05 — Multi-Container

Docker Compose

Docker Compose is a tool for defining and running multi-container Docker applications using a YAML file.

CommandDescription
docker compose up -dStart all services defined in docker-compose.yml in the background
docker compose downStop and remove containers, networks, and volumes
docker compose logs -fFollow log output from all services

Example docker-compose.yml file:

version: '3.8'

services:
  web:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html

  api:
    build: .
    ports:
      - "3000:3000"
    depends_on:
      - db

  db:
    image: postgres:13
    environment:
      POSTGRES_PASSWORD: secret
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:
