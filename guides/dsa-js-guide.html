Classes & OOP

JavaScript introduced the class keyword in ES6. It's perfectly suited for building custom data structures like Tries, Segment Trees, or Linked Lists, and feels very similar to C++ classes/structs.

C++ — Classes / Structs
struct Node {
  int val;
  Node* next;
  Node(int v) : val(v), next(nullptr) {}
};

class TrieNode {
public:
  TrieNode* children[26] = {nullptr};
  bool isWord = false;
  
  void markAsWord() {
    isWord = true;
  }
};
JavaScript — Classes
class Node {
  constructor(v) {
    this.val = v;
    this.next = null; // no pointers, just references
  }
}

class TrieNode {
  constructor() {
    this.children = new Array(26).fill(null);
    this.isWord = false;
  }
  
  // Methods don't need 'function' keyword
  markAsWord() {
    this.isWord = true;
  }
}
ℹ
The this Keyword is Mandatory
In C++, you can access class properties directly (e.g. isWord = true;). In JavaScript, you must prefix properties with this. inside methods (e.g. this.isWord = true;), otherwise it looks for a local or global variable.

Integer Division
In C++, 7 / 2 = 3. In JS, 7 / 2 = 3.5. Always use Math.floor(a / b) or ~~(a / b) for integer division. Watch out in DP and index calculations.

Declaring Variables
Use const for values you don't reassign (objects/arrays can still be mutated), let for everything else. Avoid var entirely — it has function scope, not block scope.

Maps & Sets

JS has both Map/Set (hash-based, like unordered_map) and plain objects. There is no built-in ordered map like C++ map. Use a sorted array + binary search, or a custom BST, for ordered access.

C++ unordered_map

mp[key] = val;
mp[key]++;
if(mp.count(key)) { ... }
mp.erase(key);
for(auto& [k, v] : mp) { ... }
JavaScript Map
const mp = new Map();
mp.set(key, val);
mp.set(key, (mp.get(key) ?? 0) + 1);
if (mp.has(key)) { ... }
mp.delete(key);
for (const [k, v] of mp) { ... }
C++ unordered_set
unordered_set<int> st;
st.insert(x);
st.erase(x);
if(st.count(x)) { ... }
st.size();
JavaScript Set
const st = new Set();
st.add(x);
st.delete(x);
if (st.has(x)) { ... }

⚠
Object keys vs Map keys
Plain objects {} only support string/Symbol keys. Map supports any type as key (objects, arrays, numbers). For DSA always prefer Map over {}. Also, {} has prototype keys that can collide ("constructor", "hasOwnProperty"). Use Object.create(null) if you must use objects as maps.

06 — Data Structures

Stack & Queue

No separate stack/queue classes. Arrays serve as stacks natively. For a proper O(1) queue, use a class with head pointer, not shift().

C++ stack
stack<int> st;
st.push(x);
int top = st.top(); st.pop();
st.empty();
JS Array as Stack
const st = [];
st.push(x);
const top = st[st.length - 1]; st.pop();
st.length === 0;
C++ queue
queue<int> q;
q.push(x);
int front = q.front(); q.pop();
q.empty();
JS O(1) Queue
class Queue {
  constructor() { this.data = {}; this.head = this.tail = 0; }
  push(x) { this.data[this.tail++] = x; }
  pop()  { return this.data[this.head++]; }
  front(){ return this.data[this.head]; }
  size() { return this.tail - this.head; }
  isEmpty(){ return this.size() === 0; }
}
⚠
array.shift() is O(n)
Using a plain array and calling shift() for dequeue is O(n). In BFS over large graphs, this tanks performance. Use the Queue class above, or index into the array with a pointer instead of shifting.
// BFS with O(1) queue using pointer trick
const q = [start];
let head = 0;
while (head < q.length) {
  const node = q[head++]; // O(1) dequeue
  for (const nb of adj[node]) {
    if (!visited[nb]) { visited[nb] = true; q.push(nb); }
  }
}
07 — Data Structures

Heap / Priority Queue

JavaScript has no built-in heap. You must implement one. Here's a production-ready MinHeap with the standard interface — extend for MaxHeap by negating values.

class MinHeap {
  constructor(compareFn = (a, b) => a - b) {
    this.heap = [];
    this.cmp = compareFn; // negative = a has higher priority
  }
  push(val) {
    this.heap.push(val);
    this._bubbleUp(this.heap.length - 1);
  }
  pop() {
    const top = this.heap[0];
    const last = this.heap.pop();
    if (this.heap.length > 0) { this.heap[0] = last; this._siftDown(0); }
    return top;
  }
  peek() { return this.heap[0]; }
  size() { return this.heap.length; }
  _bubbleUp(i) {
    while (i > 0) {
      const p = (i - 1) >> 1;
      if (this.cmp(this.heap[i], this.heap[p]) < 0) {
        [this.heap[i], this.heap[p]] = [this.heap[p], this.heap[i]];
        i = p;
      } else break;
    }
  }
  _siftDown(i) {
    const n = this.heap.length;
    while (true) {
      let best = i, l = 2*i+1, r = 2*i+2;
      if (l < n && this.cmp(this.heap[l], this.heap[best]) < 0) best = l;
      if (r < n && this.cmp(this.heap[r], this.heap[best]) < 0) best = r;
      if (best === i) break;
      [this.heap[i], this.heap[best]] = [this.heap[best], this.heap[i]];
      i = best;
    }
  }
}

// MaxHeap: negate the comparator
const maxHeap = new MinHeap((a, b) => b - a);

// Heap with pairs (Dijkstra): [dist, node]
const pq = new MinHeap((a, b) => a[0] - b[0]);
pq.push([0, src]);
ℹ
Dijkstra / K-th Largest Patterns
The custom comparator approach is fully flexible. Use (a,b) => a[0]-b[0] for pairs sorted by first element, or use any field in objects. Copy this class into your solutions template.

Sorting — The Biggest JS Gotcha

JavaScript's built-in .sort() converts elements to strings by default. This is the #1 source of bugs when switching from C++.

✗
[10, 9, 2, 1].sort() → [1, 10, 2, 9]
Default sort is lexicographic (string-based). Always pass a comparator for numbers. This will not throw an error — it silently gives wrong results.
C++
// Ascending
sort(v.begin(), v.end());

// Descending
sort(v.begin(), v.end(), greater<int>());

// Custom comparator
sort(v.begin(), v.end(), [](auto& a, auto& b){
  return a.first < b.first;
});
JavaScript
// Ascending — ALWAYS pass comparator
arr.sort((a, b) => a - b);

// Descending
arr.sort((a, b) => b - a);

// Custom: sort by second element
arr.sort((a, b) => {
  if (a[0] !== b[0]) return a[0] - b[0];
  return a[1] - b[1];
});
⚠
.sort() mutates in place and returns the same array
Unlike C++, arr.sort() modifies the original array. If you need to sort without mutating, spread first: [...arr].sort((a,b) => a-b). JS sort is guaranteed stable (V8 uses TimSort) since Node 11 / Chrome 70.
// No lower_bound / upper_bound — implement yourself
function lowerBound(arr, target) {
  let lo = 0, hi = arr.length;
  while (lo < hi) {
    const mid = (lo + hi) >> 1;
    if (arr[mid] < target) lo = mid + 1;
    else hi = mid;
  }
  return lo; // first index >= target
}

function upperBound(arr, target) {
  let lo = 0, hi = arr.length;
  while (lo < hi) {
    const mid = (lo + hi) >> 1;
    if (arr[mid] <= target) lo = mid + 1;
    else hi = mid;
  }
  return lo; // first index > target
}
09 — Algorithms

Binary Search

Same logic as C++, slightly different syntax. Key difference: use (lo + hi) >> 1 for mid (avoids overflow — though JS numbers are floats, so overflow is less of an issue). No lower_bound, you write it.

// Classic binary search
function binarySearch(arr, target) {
  let lo = 0, hi = arr.length - 1;
  while (lo <= hi) {
    const mid = (lo + hi) >> 1;
    if      (arr[mid] === target) return mid;
    else if (arr[mid] < target)  lo = mid + 1;
    else                         hi = mid - 1;
  }
  return -1;
}

// Binary search on answer (predicate-based)
function bsOnAnswer(lo, hi, predicate) {
  while (lo < hi) {
    const mid = Math.floor((lo + hi) / 2);
    if (predicate(mid)) hi = mid;    // find first true
    else                lo = mid + 1;
  }
  return lo;
}
10 — Algorithms

Common Patterns

Two Pointers, Sliding Window, and Prefix Sum — identical logic to C++, just JS syntax.

// ── Two Pointers ──────────────────────────────
function twoSum(arr, target) {
  let l = 0, r = arr.length - 1;
  while (l < r) {
    const s = arr[l] + arr[r];
    if      (s === target) return [l, r];
    else if (s < target)  l++;
    else                   r--;
  }
  return [-1, -1];
}

// ── Sliding Window (variable) ─────────────────
function longestSubarrayAtMostK(arr, k) {
  let l = 0, sum = 0, best = 0;
  for (let r = 0; r < arr.length; r++) {
    sum += arr[r];
    while (sum > k) sum -= arr[l++];
    best = Math.max(best, r - l + 1);
  }
  return best;
}

// ── Prefix Sum ────────────────────────────────
function buildPrefix(arr) {
  const pre = new Array(arr.length + 1).fill(0);
  for (let i = 0; i < arr.length; i++) pre[i+1] = pre[i] + arr[i];
  return pre;
}
const rangeSum = (pre, l, r) => pre[r+1] - pre[l]; // [l, r] inclusive
11 — Algorithms

Graphs

Build adjacency lists with arrays of arrays, or Map for non-integer node labels. DFS/BFS structure is identical to C++.

// Adjacency list — integer nodes
const adj = Array.from({length: n}, () => []);
for (const [u, v, w] of edges) {
  adj[u].push([v, w]);
  adj[v].push([u, w]); // undirected
}

// DFS (iterative, same as C++ with explicit stack)
function dfs(start, adj, n) {
  const visited = new Array(n).fill(false);
  const stack = [start];
  visited[start] = true;
  while (stack.length) {
    const u = stack.pop();
    for (const [v] of adj[u]) {
      if (!visited[v]) { visited[v] = true; stack.push(v); }
    }
  }
}

// Dijkstra — needs MinHeap (from heap section above)
function dijkstra(src, adj, n) {
  const dist = new Array(n).fill(Infinity);
  dist[src] = 0;
  const pq = new MinHeap((a, b) => a[0] - b[0]);
  pq.push([0, src]);
  while (pq.size()) {
    const [d, u] = pq.pop();
    if (d > dist[u]) continue;
    for (const [v, w] of adj[u]) {
      if (dist[u] + w < dist[v]) {
        dist[v] = dist[u] + w;
        pq.push([dist[v], v]);
      }
    }
  }
  return dist;
}

// Union-Find (DSU)
class DSU {
  constructor(n) {
    this.parent = Array.from({length: n}, (_, i) => i);
    this.rank = new Array(n).fill(0);
  }
  find(x) {
    if (this.parent[x] !== x) this.parent[x] = this.find(this.parent[x]);
    return this.parent[x];
  }
  union(x, y) {
    let px = this.find(x), py = this.find(y);
    if (px === py) return false;
    if (this.rank[px] < this.rank[py]) [px, py] = [py, px];
    this.parent[py] = px;
    if (this.rank[px] === this.rank[py]) this.rank[px]++;
    return true;
  }
}

Dynamic Programming & Memoization

Memoization in JS is elegant using closures and Map. Tabulation (bottom-up) is identical to C++.

// Top-down memoization with Map
function solve(n) {
  const memo = new Map();
  function dp(i, j) {
    const key = `${i},${j}`;
    if (memo.has(key)) return memo.get(key);
    let res = 0;
    // ... base cases and transitions ...
    memo.set(key, res);
    return res;
  }
  return dp(0, 0);
}

// Bottom-up (same as C++)
const dp = new Array(n + 1).fill(0);
dp[0] = 1;
for (let i = 1; i <= n; i++) {
  dp[i] = dp[i-1] + dp[i-2]; // fibonacci example
}

// 2D DP table
const dp2 = Array.from({length: n+1}, () => new Array(m+1).fill(0));
⚠
Stack Overflow in Deep Recursion
JS call stack is much smaller than C++ (typically ~10,000 frames). For n=10⁵+ recursive calls, convert to iterative or increase stack size in Node.js: node --stack-size=65536 solution.js. Alternatively, convert to explicit stack-based DFS.

const popcount = x => {

Bitwise ops in JS work on 32-bit signed integers. 1 << 31 = -2147483648 and 1 << 32 = 1 (wraps). For bitmasks beyond 30 bits, use BigInt: 1n << 40n.
